# 사립유치원 현금출납부 목별 검토 도구

현금출납부 Excel 파일을 브라우저에서 읽어, 2026 사립유치원 예·결산 및 적립금 업무처리 지침 기준으로 목 부적정 의심 항목을 선별하는 무료 정적 웹사이트입니다.

## 이번 수정본의 변경사항

- 예·결산 지침 업로드 단계 제거
- 현금출납부 `.xlsx` 또는 `.xls` 파일만 업로드하면 검토 실행
- 2026 지침 기준을 `rules.js` 내장 규칙으로 반영
- `보수`, `수리`, `구입`, `재료`, `교육비` 관련 오탐 완화
- 교원보수·직원보수는 인건비성 지출로 우선 판단
- 시설보수·개보수·공사 등은 시설비성 지출로 판단

## 주요 기능

- 현금출납부 `.xlsx` 파일 업로드
- 제목, 적요, 세부항목명, 원가비목 기반 의심 항목 자동 분류
- `목 부적정 의심`, `추가 확인 필요`, `과목경정·반납 확인` 구분
- 결과표 화면 조회
- 검토결과 `.xlsx` 및 `.csv` 다운로드

## 중요한 한계

이 도구는 AI 판단 도구가 아니라 규칙 기반 1차 선별 도구입니다.

- 제목만 보고 판단하므로 확정 지적 자료로 사용하면 안 됩니다.
- 품의서, 지출결의서, 거래명세서, 계약서, 검수조서 등 증빙 확인이 필요합니다.
- 유치원별 현금출납부 열 이름이 다르면 자동 탐색 정확도가 낮아질 수 있습니다.
- 지침이 변경되면 `rules.js`의 기준도 함께 수정해야 합니다.

## GitHub + Cloudflare Pages 배포 방법

1. GitHub Repository 루트에 `index.html`, `styles.css`, `rules.js`, `app.js`, `README.md`를 업로드합니다.
2. Cloudflare Pages에서 해당 Repository를 연결합니다.
3. 빌드 설정은 다음과 같이 둡니다.
   - Framework preset: None 또는 Static HTML
   - Build command: 비워둠
   - Build output directory: `/` 또는 비워둠
4. Deploy를 누릅니다.

## 보안 메모

- 업로드한 현금출납부 파일은 서버로 전송하지 않고 브라우저 안에서 처리합니다.
- 현재 배포본은 SheetJS를 CDN에서 불러옵니다.
- 내부 업무망 또는 보안이 중요한 환경에서는 SheetJS 파일을 내려받아 프로젝트 폴더에 포함하고, `index.html`의 CDN 주소를 로컬 파일 경로로 바꾸는 방식을 권장합니다.

## 규칙 수정 방법

`rules.js`의 `AUDIT_RULES` 배열을 수정하면 됩니다.

```js
{
  id: "example_rule",
  type: "expense",
  expected: "권장 목",
  severity: "check",
  keywords: ["제목에 포함될 단어"],
  currentNegative: ["현재 목에 포함되면 의심할 단어"],
  avoid: ["이 단어가 있으면 제외할 단어"],
  opinion: "검토 의견"
}
```

`severity` 값은 다음 중 하나를 사용합니다.

- `warn`: 목 부적정 의심
- `check`: 추가 확인 필요
- `adjust`: 과목경정·반납 확인
